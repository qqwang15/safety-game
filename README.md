## 🚀 快速开始

### 方法一：直接打开（推荐测试用）

双击 `index.html` 文件即可在浏览器中运行。

### 方法二：本地服务器（推荐正式使用）

```bash
# 进入项目目录
cd safety-game

# 使用 Python 启动服务器（需要 Python 3.x）
python -m http.server 8080

# 或使用 Node.js
npx serve .

# 然后在浏览器访问
http://localhost:8080
```

## 🖼️ 替换场景图片（重要！）

当前游戏使用的是**占位图**，你需要将它们替换为 AI 生成的真实场景图片。

### 图片要求：
- **格式**：JPG 或 PNG
- **尺寸建议**：800×600 像素以上（保持 4:3 比例）
- **文件大小**：建议 < 500KB（优化加载速度）
- **命名规则**：`level1.jpg`, `level2.jpg` ... `level5.jpg`


## 🎯 游戏玩法

1. **选择主题和关卡**
   - 首页点击"开始挑战"
   - 选择"信息安全"或"地面安全"
   - 点击已解锁的关卡进入

2. **找茬操作**
   - 在限时内点击图中你认为存在安全隐患的位置
   - ✅ 找到隐患：+100分
   - ❌ 点错位置：-20分
   - 找到 70% 以上隐患点即可通关并解锁下一关

3. **辅助功能**
   - 💡 提示按钮：高亮显示一个未找到的隐患（每关有限次数）
   - ⏸️ 暂停按钮：暂停计时器

4. **结算与学习**
   - 游戏结束后查看得分、星级评价
   - 浏览每个隐患点的**正确做法科普卡片**
   - 可选择重玩或进入下一关

## 🔧 自定义配置

### 修改隐患点位置

如果更换了新图片，需要调整隐患点的坐标位置。

编辑文件：`js/levels.js`

```javascript
hazards: [
  {
    id: 'info_1_01',
    name: '隐患名称',
    position: { 
      xPercent: 50,    // 距离左边 50%（横向位置）
      yPercent: 30,    // 距离顶部 30%（纵向位置）
      radiusPercent: 5  // 判定半径（图片宽度的5%）
    },
    correctAction: '正确做法说明',
    riskLevel: 'high', // high / medium / low
    category: '分类',
    tip: '一句话提示'
  }
]
```

**坐标定位技巧：**
1. 用图片编辑软件打开你的场景图
2. 使用标尺工具测量隐患点位置
3. 计算：xPercent = (X坐标 / 图片宽度) × 100
4. 计算：yPercent = (Y坐标 / 图片高度) × 100

### 修改游戏规则

编辑文件：`js/levels.js` 开头的 `GAME_CONFIG.rules`

```javascript
rules: {
  scorePerFound: 100,     // 每找到一个的分数
  penaltyPerWrong: -20,   // 点错的扣分
  passThreshold: 0.7,     // 通关阈值（70%）
  starThresholds: [0.6, 0.75, 0.9, 1.0], // 星级阈值
  timeBonusPerSecond: 5   // 时间奖励系数
}
```

## 🔒 数据存储

游戏进度保存在浏览器的 localStorage 中：
- 关卡解锁状态
- 最佳成绩记录
- 星级评价

**清除数据**：在浏览器控制台执行：
```javascript
localStorage.removeItem('safetyGame_levelProgress');
```
然后刷新页面即可重置所有进度。

## 🐛 问题排查

**问题1：图片无法显示**
- 检查图片路径是否正确
- 确认图片文件名是否匹配（区分大小写）
- 使用本地服务器而非直接打开文件

**问题2：点击无反应**
- 确保图片完全加载完成
- 检查浏览器控制台是否有错误
- 尝试刷新页面

**问题3：移动端操作不便**
- 缩放页面至合适比例
- 使用横屏模式获得更好体验
